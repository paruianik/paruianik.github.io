Langchain Based Multi-Model RAG System for Context Aware Question Answering
Project Description

OmniRAG is multi model Retrival Augmented Generation pipeline that allows a user to upload any PDFs, images, videos or URLs for AI based context aware question answering. Designed a very responsive and interactive UI for a better and smooth user experience. The user can choose different types of document format to upload including Pdfs, images, videos, and web pages. After submitting the query the system takes a few seconds to generate the accurate, precise and context aware answer that is shown on the UI.

The project involves many steps. The system combines document ingestion, document splitting, semantic retrival, vector embedding, Large Language Model (LLM) integration to generate an accurate and context aware question answer based on user provide content. As this is a multi modal RAG system it can handle differnt types of document format and can process all of them.

Features

The primary features of this project are:

  • Upload PDF documents and interact with their content through context-aware question answering.
  • Extract textual information from images and generate accurate responses using AI-powered analysis.
  • Analyze and understand web pages from user-provided URLs, enabling semantic retrieval and contextual question answering.
  • Convert video contents into text transcripts and perform context-aware analysis to answer user query.
  • Splite the extracted information into smaller chunks for efficient processing and context retrieval.
  • Perform semantic retrieval using vector embeddings to identify the most relevant information.
  • Integrated Groq hosted LLM with custom prompt template to generate context-aware responses.
  • Built RESTful APIs with FastAPI and developed an interactive frontend using HTML, CSS, JavaScript.

Workflow

The following workflow illustrates the end-to-end architecture of OmniRAG, a multimodal Retrieval-Augmented Generation (RAG) system. The application enables users to upload different types of content and interact with them through natural language queries. The uploaded content is processed, transformed into semantic vector embeddings, and stored for efficient retrieval. When a user submits a question, the system retrieves the most relevant contextual information and leverages a LLM to generate an accurate, context-aware response. This pipeline combines document preprocessing, semantic search, prompt engineering, and LLM inference to deliver an intelligent question-answering experience.

  • User choose what kind of document they want to upload.
  • User upload a document format including Pdfs, images, videos or any web contents.
  • Used Unstructured for extract and preprocess data from any formats of document.
  • Split the data into smaller chunks for better data preprocessing.
  • Used Hugging Face embedding models to generate semantic vector embeddings for efficient contextual retrieval.
  • Used this vector database to retrive the most relevent information (chunk) according to the user's query.
  • The task of retriver is to use these documents that matches the user query.
  • Used Groq API KEY to load a low latency Large Language Model (LLM) inference for context aware question answering.
  • Used similarity search to retrive the k most relevant chunks for the vector database.
  • Used a custom prompt template to generate a precise, accurate and structured answer.
  • Build a Langchain based RAG pipeline using LCEL(Langchain Expression Language) to generate AI based responses.

Why used HuggingFaceEmbedding for vector embedding ?

There are multiple reasons why HuggingFaceEmbedding is used for vector embedding than any other model.

  • It provides high-quality semantic embeddings for better contextual retrieval.
  • HuggingFaceEmbedding has a large collection of pre-tained embedding model for different NLP tasks.
  • It is an open source platform and free to use compared to many paid embedding API
  • It can run locally without depending on any external cloud service.
  • It easily integrates with LangChain, Chroma, FAISS, and other vector databases.
  • It Supports multiple language, so using with different languages does not cause any problem.

Why use Groq?

It provides extremely low-latency LLM inference for faster response generation. It is very much faster than many traditional cloud-hosted LLM APIs. Not only that it also supports open-source models like Meta Llama models. its easily integrates with langchain which makes it for convient to use.
